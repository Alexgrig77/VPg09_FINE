# Техническое задание FISH-MVP

## 1. Название проекта

FISH-MVP: База разрешений на рыболовство

## 2. Описание проекта

Веб-приложение для учёта разрешений на рыболовство, справочника видов рыбы и учёта выловов. Позволяет вести учёт по разрешениям, подсчитывать статистику и экспортировать данные в CSV. Минимально жизнеспособный продукт (MVP) для демонстрации и сдачи выпускного проекта.

## 3. Целевая аудитория и её боли

**Аудитория:** малые рыболовные хозяйства, инспекторы, учётчики.

**Боли:**
- Учёт разрешений и выловов ведётся в Excel или на бумаге
- Нет единого доступа к данным
- Сложно получать сводную статистику
- Требуется простая система без затрат на лицензии

## 4. Функциональные требования

1. **Авторизация**
   - Вход по логину и паролю
   - Роли: admin, user
   - Защита страниц от неавторизованного доступа

2. **Разрешения**
   - Просмотр списка, поиск
   - Добавление, редактирование, удаление
   - Поля: номер, год, район промысла, ответственный, групповое наименование, лимит

3. **Рыба**
   - Справочник видов рыбы
   - CRUD, поиск
   - Поля: наименование, групповое наименование, цена

4. **Выловы**
   - Учёт выловов по разрешениям
   - CRUD с привязкой к разрешению и виду рыбы
   - Авторасчёт суммы по количеству и цене

5. **Статистика**
   - Общее количество выловов и сумма
   - Статистика по разрешениям

6. **Экспорт**
   - Экспорт выловов в CSV

## 5. Ограничения

- Срок: MVP за 2 недели
- БД: SQLite (без отдельного сервера)
- Backend: Flask
- Без PWA, без сложных Excel-отчётов на первом этапе

## 6. Риски

- Нехватка времени — фокус на минимальном функционале
- Технические ограничения SQLite при высокой нагрузке — приемлемо для MVP
