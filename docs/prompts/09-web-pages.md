# Промпт 09: Веб-страницы

**Этап:** 4.3 Frontend  
**Среда:** [ЛОКАЛЬНО]

## Контекст

API готов. Нужны HTML-страницы с формами и таблицами.

## Задача

index.html (главная с навигацией), login.html, permissions.html, fish.html, catches.html, statistics.html. Простой стиль, таблицы, формы.

## Критерии приёмки

- Можно залогиниться, увидеть данные.
- Для админа доступны добавление/редактирование/удаление.
- Для сотрудников:
  - страница `permissions.html` — только просмотр своих разрешений (кнопки добавления/редактирования/удаления скрыты),
  - страница `catches.html` — добавление вылова только по своим разрешениям,
  - выпадающий список рыбы в `catches.html` зависит от выбранного номера разрешения.
- Навигация между страницами
- Адаптивная верстка (базовая)

## Текст промпта

```
Создай веб-страницы для FISH-MVP:

1. templates/base.html — базовый шаблон с навигацией (Разрешения | Рыба | Выловы | Статистика | Выход), block content

2. templates/login.html — форма логин/пароль, POST на /api/auth/login, редирект на /

3. templates/index.html — приветствие и ссылки на разделы

4. templates/permissions.html — таблица разрешений, кнопки Добавить/Редактировать/Удалить, поиск. Загрузка через fetch /api/permissions

5. templates/fish.html — аналогично для рыбы

6. templates/catches.html — таблица выловов, форма добавления (выбор разрешения и рыбы из списков)

7. templates/statistics.html — вывод данных из /api/statistics, кнопка экспорта CSV

8. static/css/styles.css — базовые стили (таблицы, формы, кнопки)

9. В app.py добавь роуты для отдачи HTML: /, /login, /permissions, /fish, /catches, /statistics. Проверка сессии: неавторизованных редирект на /login.
"""
