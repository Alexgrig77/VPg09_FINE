# Промпт 02: Техническое задание

**Этап:** 1.2 ТЗ  
**Среда:** [ЛОКАЛЬНО]

## Контекст

Есть `docs/IDEA.md`. Эталон — FISH. Нужно составить полное техническое задание по требованиям курса (Teory.txt): название, описание, целевая аудитория, функциональные требования, ограничения, риски.

## Задача

Создать `docs/SPECIFICATION.md` со всеми обязательными разделами ТЗ.

## Критерии приёмки

- Наличие: название, описание, целевая аудитория, боли аудитории
- Функциональные требования (список что должно делать)
- Ограничения и риски

## Текст промпта

```
На основе docs/IDEA.md создай полное техническое задание docs/SPECIFICATION.md для FISH-MVP.

Обязательные разделы по курсу:
1. Название проекта
2. Описание проекта (что делает, зачем нужен)
3. Целевая аудитория и её боли
4. Функциональные требования:
   - Авторизация (admin/user)
   - CRUD: разрешения, рыба, выловы
   - Поиск по разрешениям и рыбе
   - Простая статистика (количество, суммы)
   - Экспорт в CSV
5. Ограничения: MVP за 2 недели, SQLite, Flask
6. Риски: нехватка времени, технические ограничения

Структура чёткая, пункты нумерованы. Референс домена — проект FISH (разрешения на рыболовство).
```
