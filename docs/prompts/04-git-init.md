# Промпт 04: Инициализация Git и GitHub

**Этап:** 2.2 Git init  
**Среда:** [ЛОКАЛЬНО]

## Контекст

Проект имеет структуру. Пользователь создаст новый репозиторий на GitHub. Нужно подготовить git: init, первый коммит, инструкцию по подключению remote.

## Задача

Инициализировать git, сделать первый коммит, добавить инструкцию в README или docs.

## Критерии приёмки

- `git status` показывает чистую рабочую директорию после коммита
- В docs есть краткая инструкция по `git remote add` для нового репозитория

## Текст промпта

```
Для проекта FISH-MVP:

1. Выполни git init (если ещё не инициализирован)

2. Добавь в docs/DEPLOYMENT.md раздел «Подключение к GitHub» с командами:
   git remote add origin <URL>
   git branch -M main
   git push -u origin main

3. Сделай первый коммит всех файлов с сообщением "Initial commit: FISH-MVP structure"

Не выполняй git push — пользователь сделает это после создания репозитория на GitHub.
```
